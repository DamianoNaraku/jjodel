Now if there are customized middle point the behaviour is unchanged, but if there are not middle points, some middle points will be automatically inserted to make the loop visible.

                          • looping models Models with reference loops (non-containing ones) are now supported

                                                  • 23 gennaio - Hiding elements
                                                    • Hiding edges Edges can now be hidden according to source and target vertexes and to edge type.
                                                      To hide all edges incoming/emitted by a vertex without questions, set keep-edges="false" on his root element.
                                                      Default value is false for "Object" vertex and true for every other vertex.
                                                      If keep-edges is true, you can hide a single kind of edge use one of show-<DIRECTION>-<KIND>-edges where <DIRECTION> is one of "in", "out" and <KIND> is one of "rel", "ext", "oth" where "rel" stands for relationships, "ext" stands for extends, "oth" stands for indirect, deducted edges and anything outside previous categories.
                                                      Values can be a boolean true evaluated to "+1", a boolean false evaluated to "-1" or directly any real number.If there are conflicting instructions, like if start vertex is hiding emitted edges of that kind ("-1") and end vertex is showing incoming edges of that kind (+1), the conflicting values will be summed resulting in (-1 + 1 = 0) and the edge will be displayed if the result is strictly above 0.
                                                      By default those properties have a value of "+1".GUI support will come at a later time.
                                                      • Hiding vertices Vertices can now be hidden simply by deleting their root node inside style editor.
                                                        • Loading edges Edges are loading on wrong position during initial render, it will be fixed by hovering on them or changing viewpoint. Looking for a solution.

                                                                                      • 6 giugno 2020 - Generalization / Inheritance system Huge update with many features and bugfixes.
                                                                                        • Generalization / Inheritance: Can be added through vertex contextmen.
                                                                                          Like all edges, you can change their endpoint by selecting the edge and dragging the dot near the target vertex, and you can delete them by pressing CANC/DEL key.
                                                                                          By default inherited features are hidden in m2 and displayed in m1, this behaviour can be changed using the new filtering feature described in a following point.
                                                                                          Chain of inheritances are supported. (ClassA can extend ClassB and ClassB will extend classC, so ClassA indirectly extends classC).
                                                                                          Multiple inheritance is allowed. (ClassA can directly extend ClassB and ClassC at the same time)
                                                                                          The setup of circular inheritances will be prevented, either directly or indirectly and a popup will warn the user.
                                                                                          E.g. If ClassA extends ClassB, and ClassB extends ClassC, you cannot make classC extend classA.
                                                                                          • Overriding / Polymorphism Operations can now violate name uniqueness constraint to allow overriding and polymorphism.
                                                                                            If two functions are homonym and cannot be in a polymorphism or overriding relationship they will be marked in pairs as invalid with the same dynamic color until the user makes a change to fix the violation.
                                                                                            • Shadowing Features shadowed by redeclaration on subclasses are now detected and hidden.
                                                                                              Shadowed variables can be displayed in M1 for informative purpose, but their value will not be saved (on purpose).
                                                                                              • Style editor: Inserted "Use as default style" checkbox which can only be used on customized elements. When marked the customized style of this element will be inherited by all other elements of the same type (Class, Enum, Attribute, Reference...) with the least priority. A reminder on style priorities:
                                                                                                • 1) "Own html" customized style attached to the element itself.
                                                                                                • 2) "Inherited html" customized style attached to the meta-parent of the element (eg: a M1-class instance inheriting his style from his M2-class type).
                                                                                                • 3) User-defined "default html", eg: a M1-class instance inheriting his style from another M1-class peer marked as default style with this brand new feature.
                                                                                                • 4) Hardcoded, static "default html", the one you see on "default" viewpoint.
                                                                                                If one of those style throws an exception while rendering or executing custom code injected inside the style, that faulty style will be automatically removed and the next style in priority order will be applied.Until eventually the fourth level is reached, which has been broadly tested and should never fail.
                                                                                                • Vertex gui customization: It is now possible to set up a custom filter and condition to select which childrens of a classifier / operation should be displayed.There are shortcuts to filter children based on their type or whether they are inherited or not. More info here.
                                                                                                  • Text popup improved: Textual popup notifying the user about invalid actions, errors, tooltip or automatic fixing invalid user input now cannot show duplicates at the same time.
                                                                                                    Clicking a popup will make it disappear instantly while copying his text to the clipboard (copy-paste, old feature reminder).
                                                                                                    Right-clicking a popup will now hide all same-colored notifications.
                                                                                                    Wheel-clicking a popup will now hide all notifications.
                                                                                                    • Usability: mouse shortcuts
                                                                                                      • Rightclicking a vertex will make it follow the cursor until released, ignoring measurables. Can be useful if the only visible part of a vertex / feature is a input or a measurable, as left-clicking them will not drag the vertex.
                                                                                                      • MouseWheel-clicking anywhere on the graph and start dragging will ignore any vertexes and make the graph offset follow the cursor. Can be useful on high zoom levels, huge vertexes or densely populated graph.
                                                                                                      • ContextMenu: User right-clicking input elements inside vertexes might desire both native contextmen or customized contextmen. To access the native context men, the user must rightclicking a input field inside the currently selected feature / vertex or do a slow right-click. Here are detailed rules to check which one will appear:
                                                                                                        • RightClick and drag on any child of a vertex / feature: no custom men, the whole vertex will be dragged, measurables and inputs will be ignored.
                                                                                                        • still RightClick on a input child of a selected vertex / feature: native context men.
                                                                                                        • still RightClick on any child of a un-selected vertex / feature: custom context men.
                                                                                                        • still, slow (hold for 1 sec) RightClick a input child of vertex / feature: native men
                                                                                                        • Bug: The usage of the $ and characters outside variable templates ($##likeThis$) and inside the html of custom styles might cause errors.
                                                                                                          Until bugfix use the unicode combination \u0024 for $ and \u00A3 for when possible. This bugfix will have low priority.
